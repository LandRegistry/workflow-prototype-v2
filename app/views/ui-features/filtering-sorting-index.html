How the flagging functionality works

Technical description

Standard form logic: 4x radio buttons and selections from a 'select' (dropdown) element dictate which cases in the Progress Later list are shown and in what order. The form options are submited when the 'Submit' button is selected - the page refreshes with the new filtering and/or sorting.

Filter 'None' applies no filtering, with all cases in the Progress Later list shown. When a filter is applied (and there is no change to the default sorting option) we maintain the standard ordering setting (which in the live service appears to be the oldest priority date running from the start/top to end/bottom).

Sorting set to the 'Default sorting' follows the established ordering which is currently used (and has always been used): this prepends the Progress Later list with cases which have been expedited. There is no change to the current sorting/ordering logic. As is.

Other sorting options are self explanatory and do not prioritise prepending expedited cases at the top/start of the Progress Later list (they appear within the list according to the order/sorting selected). e.g. selecting 'Priority date: Newest' puts the newest cases according to priority date at the top/start of the list, and ascending accordingly.

The selected ordering and filters are maintained/persisted for the user session. (This means the user does not need to reapply the settings). The optins are reset to their default settings ('None' and 'Default sorting') "each day (logging off, 'end of the day')" and/or when logging back into Workflow. (Potentially achieved through session data/timings).

We only show (include) this filter/sort block (form) if there is two or more cases in the Progress Later list.

When filter/sorting selections are in action, a warning message is included within the page (below the filter/sorting options). This is contextual and is either of the following three:

  • This list has been sorted
  • This list has been filtered
  • This list has been filtered and sorted

The warning messages use the standard GDS library 'Warning Text' component (with a HMLR style class added to the parent element).

The case ordering of the design mock-ups may not reflect the form settings.
